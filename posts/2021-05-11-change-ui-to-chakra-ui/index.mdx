## 概要の前に…

ほんまにブログ書かないですね…  
「かきもの」が苦手なのと、そもそも更新すること自体がレアなので当たり前っちゃ当たり前なんですが…  
なにはともあれ、思いついたときに書くのが吉です。はい。  
せっかく環境を整備したので、継続していきたいですね。

## 概要

やることはタイトルどおりなのですが、そもそもなぜ [Chakra-UI](https://chakra-ui.com/) にたどり着いたのか。  
紆余曲折がありました…

### 1. Material-UI のスタイリングが書きづらい。

ちょっと拡張しようとすると、スタイルを拡張して記載しないといけない場合がありました。  
[Material-UI](https://material-ui.com/) には CSS in JS として [styled-components](https://styled-components.com/) が採用されているのですが、TS の補完などが効かず書きづらい部分がちらほら…

### 2. CSS in JS や CSS フレームワークとの遭遇

そんなとき、「[Tailwind CSS](https://tailwindcss.com/) なる CSS フレームワークが神！」なる噂が。  
最初は（Bootstrap みたいな、UI フレームワークか？）とも思いましたが、どうやら CSS のパーツを組み合わせて組み立てていくフレームワークのよう。  
１からコンポーネントを作成する場合にはとても有用ですが、流石に手間です。  
（ちなみに一度使って作成はしていました。その中で、CSS in JS である [Emotion](https://emotion.sh/docs/introduction) を組み合わせた [twin.macro](https://github.com/ben-rogerson/twin.macro) なるものも使ってみました。タグ内にスタイルを記載できるのは有用で、補完も強力なためかなり便利でしたが、やはり面倒です…）

### 3. Chakra-UI の発見

どうにか脱 Material-UI できないかと模索していたところ、Chakra-UI を発見しました。  
別のプロジェクトで使用してみたところ、Emotion ライクでめちゃくちゃ使いやすかったのですぐに採用しました。  
（特にダークテーマ、アクセシビリティへの配慮は気に入りました）
